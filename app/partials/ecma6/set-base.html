Classical Set collection

  • When constructing set, value must be iterable. Duplicated values won't be added
  • Key equality is based on '===' operator. And NaN is matched to NaN even though NaN !== NaN
    • var object1 = {
	name : 'Slaven'
},
object2 = {
	surname : 'Tomac'
},
object3 = {
	surname : 'Tomac'
},
mySet = new Set([object1, 'StringObject', 2, object2, object1]);
  • .size property which hold size of set
    • AppLogger.info(mySet.size);
  • .has(key) returns true/false if object exists
    • AppLogger.info(mySet.has(object1));
AppLogger.info(mySet.has('myKey1'));
AppLogger.info(mySet.has(object3));
  • .delete(key) method deletes object and returns true/false based if object existed or not
    • AppLogger.info(mySet.get(object1));
mySet.delete(object1);
AppLogger.info(mySet.get(object1));
  • .entries() returns iterator for iterating objects (see Generators for more details how iterators work).
    • var iterator = mySet.entries();
AppLogger.info(iterator.next());
AppLogger.info(iterator.next());
AppLogger.info(iterator.next());
  • .forEach() returns iterator over values in set
    • mySet.forEach(function(val) {
	AppLogger.info(val);
});
  • .add(object) adds object to Set. Returns Set object back
  • AppLogger.info(mySet.has(object3));
mySet.add(object3);
AppLogger.info(mySet.has(object3));
    .keys() returns iterator over values (same as .values)
    var keysIterator = mySet.keys();
AppLogger.info(keysIterator.next());
AppLogger.info(keysIterator.next());
AppLogger.info(keysIterator.next());
    .values() returns iterator over values (same as keys)
    var valuesIterator = mySet.values();
AppLogger.info(valuesIterator.next());
AppLogger.info(valuesIterator.next());
AppLogger.info(valuesIterator.next());
    Set object has default iterator which is used in for..of iterator. It iterates through objects stored in Set (see Generators-for..of for more details about default iterators)
    for (var val of mySet) {
	AppLogger.info('val');
	AppLogger.info(val);
}
