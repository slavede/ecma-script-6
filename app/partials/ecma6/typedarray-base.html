Allows declaring array with specicic type, length and/or buffer.

  • Few constructors:
    • new TypedArray(length)
    • new TypedArray(typedArray)
    • new TypedArray(object)
    • new TypedArray(buffer [,bytesOffset [,length]])
    where
    • length - if given, array with that many zeroes will be created
    • typedArray - it can be object of any of the typed arrays, copies (and converts) values into new array
    • object - creates array from the object
    • TypedArray can be one of the following:
      • Int8Array()
      • Uint8Array()
      • Uint8ClampedArray() - values clamped to 0-255
      • Int16Array()
      • Uint16Array()
      • Int32Array()
      • Uint32Array()
      • Float32Array()
      • Float64Array()
  • They don't support .push()!!! :o

Adding/converting elements

  • Initially it's an array with zeroes 
    var int8Array = new Int8Array(8);
AppLogger.info(int8Array.join(','));
  • To add one element you can't use push as in normal array types, but assign it via index access 
    int8Array[0] = 1;
AppLogger.info(int8Array.join(','));
int8Array[1] = -123;
AppLogger.info(int8Array.join(','));
int8Array[1] = -128;
AppLogger.info(int8Array.join(','));
  • If assigned value out of the scope it will be modulo modified 
    int8Array[1] = -129;
AppLogger.info(int8Array.join(','));
int8Array[4] = -130;
AppLogger.info(int8Array.join(','));
int8Array[5] = 129;
AppLogger.info(int8Array.join(','));
  • If assigning float to int, it will just strip off everything behind decimal point 
    int8Array[6] = 1.45;
AppLogger.info(int8Array.join(','));
int8Array[7] = 1.55;
AppLogger.info(int8Array.join(','));
  • If assigning value to index which is outside of the scope of array it will be just ignored. 
    int8Array[8] = 1.55;
AppLogger.info(int8Array.join(','));
